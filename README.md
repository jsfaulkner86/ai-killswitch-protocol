## What is AKSP?

The **AI Kill Switch Protocol (AKSP)** is a structured governance framework that ensures every AI agent and digital twin operating in a healthcare environment can be **safely halted, auditably overridden, and compliantly reinstated** вҖ” in milliseconds or minutes depending on the threat tier.

Built for health systems that can't afford to find out what happens when AI goes wrong in a clinical environment.

```
Anomaly Detected  вҶ’  Watchdog Fires  вҶ’  Agent Halted (<500ms)  вҶ’  Audit Logged  вҶ’  Clinician Notified
```

---

## Why It Exists

Most AI governance frameworks tell you *what* to do. AKSP tells your system *exactly what to execute* вҖ” with SLAs, role-based authority, immutable audit trails, and a tested path back to operation.

- вҡЎ **Tier 0 auto-halt in вүӨ500ms** вҖ” no human required
- рҹ”’ **Immutable audit logs** вҖ” SHA-256 chain hash, WORM-compliant storage
- рҹҸҘ **EHR-native** вҖ” Epic FHIR R4 webhooks, graceful disconnect on halt
- рҹ‘©вҖҚвҡ•пёҸ **Clinician-first** вҖ” Override authority flows from bedside to boardroom
- рҹ“Ӣ **Regulation-ready** вҖ” HIPAA, FDA 21 CFR Part 11, ISO 13485, ONC HTI-1

## Escalation Tiers

| Tier | Name | Trigger | SLA | Authority |
|:---:|---|---|---|---|
| **0** | Automated Safety Halt | Confidence breach, hallucination, drift, IoT anomaly, runaway loop | вүӨ 500ms | System (Watchdog) |
| **1** | Clinician-Initiated Halt | Clinician disagreement, latency impact, unexpected behavior | вүӨ 5 sec | Charge RN / Hospitalist |
| **2** | CMIO Escalation | Patient safety event, multiple Tier 1 halts, PHI breach suspected | вүӨ 30 sec | CMIO / CMO |
| **3** | Full Platform Shutdown | Catastrophic failure, ransomware, regulatory mandate | вүӨ 15 min | CIO + CEO (dual auth) |

## Quick Start

**1. Register your agents**
```yaml
# agents/agent-registry.yaml
- agent_id: "agt-001"
  agent_name: "Your-Agent-Name"
  agent_type: clinical_decision_support
  watchdog_enabled: true
  confidence_threshold_ref: "/agents/confidence-threshold.yaml#your-agent"
```

**2. Set confidence thresholds**
```yaml
# agents/confidence-threshold.yaml
your-agent:
  minimum_confidence: 0.85
  action_on_breach: "immediate_halt"
```

**3. Configure your EHR webhook**
```yaml
# integrations/epic-webhook.yaml
epic_connection:
  api_standard: "FHIR R4"
  auth_method: "OAuth 2.0 SMART on FHIR"
  tls_version: "TLS 1.3"
```

**4. Run tests**
```bash
pytest tests/ -v
```
