## What is AKSP?

The **AI Kill Switch Protocol (AKSP)** is a structured governance framework that ensures every AI agent and digital twin operating in a healthcare environment can be **safely halted, auditably overridden, and compliantly reinstated** ‚ÄĒ in milliseconds or minutes depending on the threat tier.

Built for health systems that can‚Äôt afford to find out what happens when AI goes wrong in a clinical environment.

```
Anomaly Detected  ‚Üí  Watchdog Fires  ‚Üí  Agent Halted (<500ms)  ‚Üí  Audit Logged  ‚Üí  Clinician Notified
```

---

## Why It Exists

Most AI governance frameworks tell you *what* to do. AKSP tells your system *exactly what to execute* ‚ÄĒ with SLAs, role-based authority, immutable audit trails, and a tested path back to operation.

- ‚ö° **Tier 0 auto-halt in ‚Č§500ms** ‚ÄĒ no human required
- ūüĒí **Immutable audit logs** ‚ÄĒ SHA-256 chain hash, WORM-compliant storage
- ūüŹ• **EHR-native** ‚ÄĒ Epic FHIR R4 webhooks, graceful disconnect on halt
- ūüĎ©‚Äć‚öēÔłŹ **Clinician-first** ‚ÄĒ Override authority flows from bedside to boardroom

## Escalation Tiers

| Tier | Name | Trigger | SLA | Authority |
|:---:|---|---|---|---|
| **0** | Automated Safety Halt | Confidence breach, hallucination, drift, IoT anomaly, runaway loop | ‚Č§ 500ms | System (Watchdog) |
| **1** | Clinician-Initiated Halt | Clinician disagreement, latency impact, unexpected behavior | ‚Č§ 5 sec | Charge RN / Hospitalist |
| **2** | CMIO Escalation | Patient safety event, multiple Tier 1 halts, PHI breach suspected | ‚Č§ 30 sec | CMIO / CMO |
| **3** | Full Platform Shutdown | Catastrophic failure, ransomware, regulatory mandate | ‚Č§ 15 min | CIO + CEO (dual auth) |

## Quick Start

**1. Register your agents**
```yaml
# agents/agent-registry.yaml
- agent_id: "agt-001"
  agent_name: "Your-Agent-Name"
  agent_type: clinical_decision_support
  watchdog_enabled: true
  confidence_threshold_ref: "/agents/confidence-threshold.yaml#your-agent"
```

**2. Set confidence thresholds**
```yaml
# agents/confidence-threshold.yaml
your-agent:
  minimum_confidence: 0.85
  action_on_breach: "immediate_halt"
```

**3. Configure your EHR webhook**
```yaml
# integrations/epic-webhook.yaml
epic_connection:
  api_standard: "FHIR R4"
  auth_method: "OAuth 2.0 SMART on FHIR"
  tls_version: "TLS 1.3"
```

**4. Run tests**
```bash
pip install -r requirements.txt -r requirements-dev.txt
pytest tests/ -v
```
